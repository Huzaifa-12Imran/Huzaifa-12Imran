## 🧠 The Developer Behind the Screen

> *"Most devs pick one lane. I build the web and render the world."*

I exist at an intersection that most developers never explore — where **web architecture** meets **3D world-building**. By day I'm wiring REST APIs and designing component trees; by night I'm sculpting game environments in Blender and scripting real-time interactions in Unreal Engine 5.

What that means for you: I don't just build *functional* — I build *experiential*. Every project I touch gets a layer of UX thinking that most backend devs skip, and a systems-level pragmatism that most designers never consider.

**The short version:**
- 🏗️ Full-stack web apps with clean architecture and a strong UX backbone (MERN)
- 🌍 Immersive 3D environments and game experiences (Unreal Engine 5 + Blender)
- ⚙️ Comfortable from frontend down to C++ and Assembly — the whole stack, including the metal
- 🤝 Actively open to internships, freelance gigs, and full-time roles

## 🧩 The Unconventional Part of My Stack

Here's what rarely shows up on profiles but defines how I think:

**Unreal Engine 5** — I've worked with Lumen lighting, Nanite geometry, and Blueprint scripting. This isn't just a hobby. Understanding real-time rendering makes me think about frontend performance *very* differently.

**Blender** — Modeling, rigging, and rendering. When I design UI components, I'm thinking about shadow, depth, and material — not just hex codes.

**Assembly (x86)** — I can read it, write small programs in it, and debug memory at that level. It makes me take memory management seriously in every language I use.

**Logo & Graphic Design** — I've done branding work. I know what a kerning mistake looks like and why it matters. Design is not decoration.
